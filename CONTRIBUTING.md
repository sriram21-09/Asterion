# Contributing to Asterion

First off, thank you for considering contributing to Asterion! It's people like you that make Asterion a great tool for explainable telecom localization and investigation support.

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues first.
* Use a clear and descriptive title.
* Describe the exact steps which reproduce the problem.
* Include details about your environment (OS, Python version, Node version).

### Suggesting Enhancements
* Open an issue explaining the proposed change.
* Explain why this enhancement would be useful to most Asterion users.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure dependencies are managed properly.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes.
5. Format your code (e.g., using `black` for Python and `prettier` for JavaScript/TypeScript).
6. Document any change in the `CHANGELOG.md`.

## Coding Standards

### Python (Backend)
* Use PEP 8 guidelines.
* Include type hints where possible.
* Use `black` and `isort` for formatting.
* Document functions with descriptive docstrings.

### JavaScript (Frontend)
* Use functional components and hooks.
* Maintain clean state management.
* Write clean, semantic JSX.
